**版本**：1.0  
**日期**：2026-06-22  
**作者**：Sarah，Product Owner  
**需求质量评分**：92/100  
**产品方向**：模板驱动 + Skills 组合 + Dify 创建 Agent  

---

## 1. 执行摘要

七月 Agent 实战社区是一个面向 AI 开发者、AI 产品经理、AI 运营和自媒体创作者的 Agent 实战平台。产品不定位为单纯的 AI 资讯站，也不定位为纯低代码工作台，而是以 Dify 为底层 AI 应用运行能力，向上构建一个可发现、可学习、可复用、可创建 Agent 的社区型产品。

首版 MVP 的核心目标是验证 **Agent 创建转化**。用户可以从场景化 Agent 模板进入，理解模板能解决什么问题，查看关联 Skills、知识库建议和示例输出，再一键进入 Dify 创建或复制 Agent。AI 资讯、GitHub AI Top 100、Skills 技能库和七月 AI 课程都作为 Agent 创建路径的支撑入口，而不是彼此割裂的内容栏目。

产品成功的关键不在于内容数量，而在于能否让用户从“看到一个 AI 场景”快速走到“创建一个可运行的 Agent”。因此，首版优先建设 Agent 模板中心、Skills 技能库、模板详情页、Dify 创建入口、基础搜索和内容运营后台，暂缓复杂社区等级、付费交易、客户端本地系统操作等高风险能力。

## 2. 背景与问题

### 2.1 当前背景

AI 工具、开源项目、Prompt、Workflow、Agent 和内容课程增长很快，但用户经常面临三个问题：

- 信息分散：AI 资讯、GitHub 项目、工具技能、课程资源分别散落在不同平台。
- 落地困难：用户知道某个模型或项目很火，但不知道能用它解决什么业务问题。
- 创建成本高：Dify 等平台具备强大的 Agent 与 Workflow 能力，但非开发者仍需要场景模板和操作引导。

### 2.2 用户痛点

AI 开发者关注项目价值、技术实现、API、插件、工具链和可二开空间，但缺少按实际场景组织的 Agent 模板与技能组合。

AI 产品、运营和自媒体人关注增长、内容生产、工作流效率和爆款案例，但缺少低门槛的 Agent 创建路径，也难以判断某个 AI 技术如何转化为工作效率。

### 2.3 产品机会

基于 Dify 已有的应用、工作流、知识库、插件、模型接入和 API 能力，七月 Agent 实战社区可以把“内容发现”和“Agent 创建”打通，形成更短的转化路径：

```text
看到场景 -> 理解价值 -> 选择模板 -> 组合 Skills -> 绑定知识库 -> 创建 Agent -> 试运行
```

## 3. 产品定位

### 3.1 一句话定位

七月 Agent 实战社区是一个从 AI 前沿发现到 Agent 落地创建的一站式实战社区。

### 3.2 核心价值主张

- 对 AI 开发者：快速发现值得研究和复用的 AI 项目、Skills、Agent 模板和 Dify 二开入口。
- 对 AI 产品/运营/自媒体人：快速把内容生产、选题策划、爆款文案、资料整理和知识库问答转化为可运行 Agent。
- 对平台自身：以模板和 Skills 为核心资产，形成可持续运营的 Agent 场景库。

### 3.3 产品边界

首版不是完整内容社区，不追求全量社交关系链；也不是重写 Dify，不重复建设模型管理、工作流引擎和知识库底层能力。首版聚焦在 Dify 之上的场景层、模板层和内容转化层。

## 4. 目标用户

### 4.1 主要用户：AI 产品/运营/自媒体人

- **角色**：AI 产品经理、运营人员、自媒体创作者、课程运营者。
- **目标**：快速生成小红书、抖音、公众号、课程推广、产品分析等内容。
- **痛点**：会用 AI 工具，但不会系统搭建 Agent；缺少高质量模板和示例。
- **技术水平**：初级到中级。
- **首版价值**：从模板出发，一键创建内容生产类 Agent。

### 4.2 主要用户：AI 开发者

- **角色**：AI 应用开发者、全栈开发者、独立开发者、技术负责人。
- **目标**：研究 AI 项目、复用技能组件、基于 Dify 二开 Agent 应用。
- **痛点**：GitHub 项目很多，但项目作用、适用场景和实际价值不清晰。
- **技术水平**：中级到高级。
- **首版价值**：通过榜单、Skills 和模板理解技术落地路径。

### 4.3 次要用户：学习型用户

- **角色**：AI 学习者、课程学员、自由职业者。
- **目标**：学习 AI 工具和 Agent 搭建方法。
- **痛点**：学习资源多但不成体系，难以形成实战作品。
- **技术水平**：初级。
- **首版价值**：通过免费课程资源和模板案例完成入门。

## 5. 产品目标与成功指标

### 5.1 首版业务目标

首版重点验证：用户是否愿意从社区模板进入，并完成 Agent 创建。

### 5.2 核心 KPI

- Agent 模板详情页到创建点击率：首月达到 15%。
- Agent 创建流程完成率：首月达到 35%。
- 模板创建后首次试运行率：首月达到 50%。
- Skills 绑定率：使用模板创建的 Agent 中，至少 40% 绑定一个或多个 Skills。
- 7 日回访率：完成 Agent 创建用户达到 20%。

### 5.3 辅助指标

- 搜索使用率：访问用户中 25% 使用站内搜索。
- 收藏率：模板、Skills、GitHub 项目或文章详情页收藏率达到 8%。
- 课程资源点击率：课程频道访问到外部资源点击率达到 10%。
- 内容运营效率：运营人员可以在 10 分钟内发布一条模板或 Skills 内容。

## 6. MVP 范围

### 6.1 MVP 必须包含

1. **Agent 模板中心**
   - 场景化模板列表。
   - 支持分类、标签、搜索、排序。
   - 支持展示模板用途、适用人群、预期输出和关联 Skills。

2. **Agent 模板详情页**
   - 展示模板解决的问题、输入要求、输出示例、使用步骤。
   - 展示关联 Skills、推荐知识库类型和参考内容。
   - 提供“使用该模板创建 Agent”入口。

3. **Skills 技能库**
   - 支持分类、标签、简介、适用场景和示例。
   - 支持来自 `skills.sh` 的技能概念参考，但内容需要本地化整理。
   - 支持在模板详情页关联 Skills。

4. **Dify 创建入口**
   - 用户点击模板创建后，进入 Dify 应用或工作流创建流程。
   - 首版可采用跳转、复制模板配置、导入 DSL 或调用 Dify 创建接口等方式实现，具体以开发调研后的可用接口为准。

5. **内容转 Agent 入口**
   - AI 资讯、GitHub 榜单、课程资源页面都提供“相关 Agent 模板”推荐。
   - 用户可以从内容详情页进入模板详情页。

6. **基础互动**
   - 点赞、收藏、评论用于验证内容价值。
   - 首版评论只做基础文本评论和删除管理，不做复杂楼中楼。

7. **运营后台能力**
   - 支持运营人员维护模板、Skills、文章、GitHub 项目和课程资源。
   - 支持上下架、推荐位、标签和排序权重。

### 6.2 MVP 不包含

- 不做复杂用户等级、积分、勋章体系。
- 不做模板付费交易和佣金结算。
- 不做完整小红书、抖音平台账号托管发布。
- 不做客户端本地系统操作能力。
- 不做企业多租户私有社区高级权限。
- 不重写 Dify 的模型管理、工作流引擎、知识库和插件运行机制。

## 7. 核心用户路径

### 7.1 模板驱动创建路径

```text
用户进入首页
-> 浏览 Agent 模板中心
-> 选择“短视频爆款文案 Agent”
-> 查看模板介绍、输出示例、关联 Skills
-> 选择是否绑定知识库
-> 点击“使用模板创建 Agent”
-> 进入 Dify 创建或复制流程
-> 试运行并保存
```

### 7.2 内容驱动创建路径

```text
用户查看 AI 资讯或 GitHub 榜单
-> 理解某个项目或技术的实际用途
-> 页面推荐相关 Agent 模板
-> 进入模板详情
-> 创建 Agent
```

### 7.3 Skills 驱动创建路径

```text
用户进入 Skills 技能库
-> 搜索“网页搜索”“内容改写”“热点分析”
-> 查看 Skill 作用和示例
-> 查看关联 Agent 模板
-> 选择模板创建 Agent
```

## 8. 功能需求

### 8.1 首页与导航

**目标**：让用户第一屏明确这是一个能创建 Agent 的实战社区。

**功能要求**：

- 首页主入口突出“Agent 模板”和“创建 Agent”。
- 顶部导航包含：Agent 模板、Skills 技能库、AI 资讯、GitHub 榜单、AI 课程。
- 首页展示推荐模板、热门 Skills、最新 AI 资讯、GitHub 热门项目和课程资源。
- 每个内容模块都应有明确的创建 Agent 引导。

**验收标准**：

- 用户能在 3 次点击内从首页进入任意模板详情页。
- 用户能在首页看到至少 6 个 Agent 模板。
- 首页所有推荐内容均能跳转到详情页或模板创建路径。

### 8.2 Agent 模板中心

**目标**：成为用户创建 Agent 的主要入口。

**功能要求**：

- 模板列表支持分类筛选：内容创作、技术研究、运营增长、知识库问答、项目分析、课程学习。
- 支持按热度、最新、收藏数、创建转化率排序。
- 支持关键词搜索。
- 模板卡片展示：名称、简介、适用人群、标签、关联 Skills 数、使用次数、收藏数。
- 支持运营推荐标识。

**验收标准**：

- 用户可以通过关键词搜索找到模板。
- 用户可以按分类筛选模板。
- 模板卡片信息足以让用户判断是否进入详情页。

### 8.3 Agent 模板详情页

**目标**：让用户清楚知道模板能解决什么问题，并愿意创建 Agent。

**功能要求**：

- 展示模板名称、核心场景、适用人群、业务价值。
- 展示输入示例和输出示例。
- 展示关联 Skills 及其作用。
- 展示推荐知识库类型，例如产品资料库、个人选题库、GitHub 项目库、课程资料库。
- 展示创建前准备清单。
- 提供主按钮“使用模板创建 Agent”。
- 提供收藏、点赞、评论。

**验收标准**：

- 用户无需离开详情页即可理解模板用途。
- 点击创建按钮后进入 Dify 创建路径。
- 如果创建失败，需要提示原因和可重试动作。

### 8.4 Skills 技能库

**目标**：沉淀可复用的 Agent 能力组件。

**功能要求**：

- Skill 支持分类：搜索检索、内容生成、数据分析、网页浏览、文件处理、社交媒体、代码开发、知识库增强、自动化操作。
- Skill 详情包含：名称、简介、适用场景、输入参数说明、输出说明、使用示例、关联模板。
- 支持搜索、分类筛选、收藏。
- 支持从 Skill 详情进入相关 Agent 模板。

**验收标准**：

- 每个 Skill 必须有一句话说明和至少一个使用示例。
- Skill 详情页必须展示至少一个关联模板或明确提示暂无关联模板。
- 用户可以收藏 Skill。

### 8.5 AI 资讯与技术文章

**目标**：提供高信息密度的 AI 内容入口，并服务于 Agent 创建转化。

**功能要求**：

- 支持国内外 AI 资讯、技术文章、模型动态、产品更新。
- 支持搜索、标签、来源、发布时间筛选。
- 支持点赞、收藏、评论。
- 文章详情页展示相关 Skills 和相关 Agent 模板。
- 列表页信息密度要高，避免营销式大卡片堆叠。

**验收标准**：

- 用户可以按关键词搜索文章。
- 文章详情页能推荐至少一个相关模板或 Skill。
- 列表页在桌面端首屏至少展示 8 条内容。

### 8.6 GitHub AI Top 100 榜单

**目标**：帮助用户理解热门 AI 开源项目的实际作用，并转化为模板和 Skills。

**功能要求**：

- 榜单展示项目名称、Star、语言、分类、更新时间、简介。
- 详情页解释项目是什么、解决什么问题、适合谁用、实际应用场景。
- 支持标签：Agent 框架、RAG、LLM 应用、模型服务、开发工具、数据处理、自动化。
- 支持收藏和评论。
- 项目详情页展示相关 Skills 和 Agent 模板。

**验收标准**：

- 每个项目详情必须包含“实际作用”说明。
- 每个项目详情必须有“适合人群”说明。
- 用户可以从项目详情进入相关模板。

### 8.7 七月精品 AI 课程

**目标**：作为学习资源和引流入口，帮助用户从学习走向实战创建。

**功能要求**：

- 课程资源分为国内免费资源、国外资源、实战教程、工具课程。
- 国内资源课程免费分享，只做引流和学习路径补充。
- 课程详情展示课程简介、适合人群、学习目标、资源链接。
- 课程详情页推荐相关 Agent 模板。

**验收标准**：

- 课程详情页必须展示资源来源和访问入口。
- 国内免费资源不得包装为平台自有付费课程。
- 用户可以从课程详情进入相关模板。

### 8.8 搜索能力

**目标**：让用户跨内容类型找到模板、Skills、文章、项目和课程。

**功能要求**：

- 支持全站搜索。
- 搜索结果按类型分组：Agent 模板、Skills、AI 资讯、GitHub 项目、课程。
- 支持关键词高亮。
- 支持按内容类型过滤。
- 首版搜索可先采用数据库全文检索或轻量搜索方案，后续再升级向量搜索。

**验收标准**：

- 搜索结果页能在 2 秒内返回结果。
- 搜索为空时展示推荐模板和热门 Skills。
- 用户可以从搜索结果直接进入详情页。

### 8.9 互动与收藏

**目标**：验证内容价值和用户兴趣。

**功能要求**：

- 支持点赞、收藏、评论。
- 收藏对象包括模板、Skills、文章、GitHub 项目、课程。
- 用户可以在个人中心查看收藏内容。
- 评论支持运营删除和基础审核。

**验收标准**：

- 用户登录后可以收藏任意支持收藏的内容。
- 用户取消收藏后列表状态同步更新。
- 评论提交失败时给出明确错误提示。

### 8.10 运营后台

**目标**：让运营人员能持续维护模板和内容资产。

**功能要求**：

- 支持新增、编辑、上下架 Agent 模板。
- 支持新增、编辑、上下架 Skills。
- 支持维护文章、GitHub 项目、课程资源。
- 支持标签、分类、推荐位、排序权重。
- 支持查看基础数据：浏览量、收藏数、创建点击数。

**验收标准**：

- 运营人员可以不改代码发布一个新模板。
- 下架内容不再出现在前台列表和搜索结果中。
- 推荐位内容可以通过后台调整。

## 9. 与 Dify 底座的关系

### 9.1 复用能力

首版应优先复用 Dify 的以下能力：

- 用户登录和工作区能力。
- App、Workflow、Agent 创建与运行能力。
- 知识库 Dataset 能力。
- 模型供应商和 API Key 管理能力。
- 插件、工具和 Sandbox 能力。
- 应用 API 发布能力。

### 9.2 新增产品层能力

七月 Agent 社区主要新增：

- Agent 模板内容模型。
- Skills 技能内容模型。
- GitHub 项目榜单内容模型。
- AI 资讯与课程内容模型。
- 模板到 Dify 创建的映射关系。
- 内容互动和收藏。
- 社区内容运营后台。

### 9.3 推荐技术落点

- 前端新增页面建议放在 `web/app/(commonLayout)` 下，保持和 Dify 控制台一致的登录与布局体验。
- 后端建议在 `api/controllers/console` 下新增七月社区相关控制器，复用 Dify 当前 console API 鉴权模式。
- 业务逻辑建议放在 `api/services` 下独立服务模块，避免污染 Dify 核心 workflow、agent、rag 引擎。
- 数据模型建议放在 `api/models`，并通过 migration 管理。

## 11. 数据模型建议

### 11.1 AgentTemplate

- `id`：模板 ID。
- `name`：模板名称。
- `slug`：URL 标识。
- `summary`：一句话简介。
- `description`：详细说明。
- `category`：模板分类。
- `tags`：标签。
- `target_users`：适用人群。
- `input_examples`：输入示例。
- `output_examples`：输出示例。
- `recommended_dataset_types`：推荐知识库类型。
- `dify_app_template_id`：关联 Dify 应用或工作流模板 ID。

## 12. 用户故事与验收标准

### Story 1：通过模板创建 Agent

**作为** AI 运营人员，  
**我想要** 从爆款文案模板一键创建 Agent，  
**以便** 快速生成小红书或抖音内容草稿。

**验收标准**：

- 用户可以在模板中心找到内容创作类模板。
- 用户进入详情页后能看到输入示例和输出示例。
- 用户点击创建后进入 Dify 创建流程。
- 创建失败时系统提示失败原因和重试入口。

### Story 2：查看 Skill 并理解用途

**作为** AI 开发者，  
**我想要** 查看某个 Skill 的作用、参数和示例，  
**以便** 判断是否适合加入我的 Agent。

**验收标准**：

- Skill 详情页展示简介、使用场景、输入输出和示例。
- Skill 详情页展示关联模板。
- 用户可以收藏 Skill。

### Story 3：从 GitHub 项目进入 Agent 模板

**作为** AI 开发者，  
**我想要** 查看 GitHub AI Top 100 项目的实际作用，  
**以便** 找到可复用的技术方案或 Agent 模板。

**验收标准**：

- 项目详情页说明项目是什么、解决什么问题、适合谁。
- 项目详情页展示相关 Skills 或 Agent 模板。
- 用户可以从项目详情跳转到模板详情。

### Story 4：从资讯文章进入 Agent 模板

**作为** AI 产品经理，  
**我想要** 在阅读 AI 资讯时看到相关 Agent 模板，  
**以便** 把趋势信息转化为实际工具。

**验收标准**：

- 文章详情页展示相关模板推荐。
- 推荐模板至少展示名称、简介和创建入口。
- 用户可以收藏文章。

### Story 5：运营人员维护模板

**作为** 社区运营人员，  
**我想要** 在后台发布和调整 Agent 模板，  
**以便** 持续运营平台内容。

**验收标准**：

- 运营人员可以新增、编辑、发布和下架模板。
- 模板下架后前台不可见。
- 运营人员可以设置推荐位和排序权重。

## 13. 非功能需求

### 13.1 性能

- 列表页接口响应时间目标小于 800ms。
- 搜索接口响应时间目标小于 2 秒。
- 模板详情页首屏加载目标小于 2 秒。
- 首页桌面端首屏至少展示 6 个模板和 8 条内容摘要。

### 13.2 安全

- 复用 Dify 登录态和权限体系。
- 运营后台仅管理员或指定角色可访问。
- 评论内容需要基础敏感词过滤和运营删除能力。
- 外部链接需要标记来源，避免误导为平台自有内容。
- 未来涉及本地系统操作能力时，必须单独设计权限、审计和用户确认机制，不进入 MVP。

### 13.3 可运营性

- 模板、Skills、文章、项目和课程都必须支持上下架。
- 推荐位和排序权重必须可运营配置。
- 数据指标至少包括浏览量、收藏数、点赞数、创建点击数。

### 13.4 可扩展性

- 内容模型需要支持后续引入爬虫、RSS、GitHub API 或人工采编。
- 搜索能力首版可轻量实现，但模型设计需支持后续升级向量搜索。
- 模板与 Dify 应用之间应使用映射关系，不把模板内容硬编码到前端。

## 15. 分阶段路线图

### Phase 1：MVP，验证 Agent 创建转化

周期建议：3 到 5 周。

范围：

- Agent 模板中心。
- 模板详情页。
- Skills 技能库。
- Dify 创建入口。
- AI 资讯、GitHub 榜单、课程资源的基础内容页。
- 基础搜索、收藏、点赞、评论。
- 运营后台基础维护能力。

成功标准：

- 用户可以从模板完成 Agent 创建。
- 运营人员可以维护模板和 Skills。
- 关键转化漏斗数据可追踪。

### Phase 2：增强内容到 Agent 的推荐能力

周期建议：3 到 4 周。

范围：

- 文章、项目、课程与模板的自动或半自动关联。
- 搜索结果优化。
- 模板复制成功率优化。
- 更多场景模板。
- 用户个人收藏中心增强。

### Phase 3：社区与商业化探索

周期建议：4 到 8 周。

范围：

- 用户主页和贡献内容。
- 模板投稿和审核。
- 精品模板专题。
- 课程转化路径。
- 企业咨询和私有化服务线索收集。

### Phase 4：高级 Agent 能力

周期建议：独立立项。

范围：

- 模型 API 或中转服务更深接入。
- 本地客户端系统操作能力。
- 更复杂的自动化工作流。
- 多 Agent 协作模板。

## 16. 风险评估

| 风险 | 概率 | 影响 | 应对策略 |
|---|---|---|---|
| Dify 模板复制或创建接口不满足预期 | 中 | 高 | 先调研 Dify 现有 App/Workflow 创建能力，必要时首版采用跳转和配置说明过渡 |
| 内容栏目过多导致 MVP 发散 | 高 | 高 | 所有内容栏目都必须服务于 Agent 创建转化，不做独立内容大而全 |
| 非开发者仍觉得 Dify 创建流程复杂 | 中 | 高 | 模板详情页提供准备清单、默认配置和示例输入，减少理解成本 |
| GitHub 榜单数据维护成本高 | 中 | 中 | 首版允许人工运营维护，后续再接 GitHub API 自动更新 |
| Skills 概念对普通用户抽象 | 中 | 中 | Skill 页面必须用场景化语言解释，不只写技术参数 |
| 评论区内容审核压力 | 中 | 中 | 首版评论能力保持简单，提供删除、隐藏和基础过滤 |
| 后续本地系统操作能力安全风险高 | 高 | 高 | 明确排除出 MVP，单独做权限、沙箱、审计和确认机制设计 |

## 21. 术语表

- **Agent 模板**：面向特定业务场景预设的智能体配置、说明和示例集合。
- **Skill**：Agent 可复用的能力单元，例如搜索、摘要、改写、文件解析、API 调用。
- **知识库**：Dify Dataset 能力，用于为 Agent 提供私有资料检索。
- **Dify 创建入口**：用户从社区模板进入 Dify 创建或复制 Agent 的路径。
- **内容转 Agent**：用户从文章、榜单、课程等内容进入相关 Agent 模板并完成创建。
